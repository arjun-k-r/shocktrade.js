Synopsis

ShockTrade is an Online Interactive Trading Simulation and Investment Research System (stocks, ETFs, fixed income securities, and currencies). The system allows consumers to easily identify investment opportunities (that they would not have been able to discover otherwise) via a simple easy-to-use web-based interface. The system can be used to research stocks (and other investment vehicles) in near real-time (during trading hours) or historically via a user-defined date range and investment criteria.

As you may have noticed, ShockTrade makes use of a disparate mix of technologies. This is partially due to a transition from Java to Scala, MySQL (as the primary database) to MongoDB, JMS to Akka's Actor Framework, and Hibernate to Spring Data. However, my development philosophy has always been "use the right tool for the right job," as a result, MongoDB now serves as the primary database, while MySQL is being used for data with sensitivity to latency with respect to consistency. Moreover, Cassandra acts as the data repository for frequently retrieved reference data as it offers the highest performance (in a horizontal cluster) of the three databases, but at the cost of additional complexity.

Modern web applications are being built to scale to thousands (if not tens of thousands) of concurrent users. My goal for ShockTrade is to create an architecture that could achieve such scale by utilizing a web 3.0 pure-JavaScript front-end (via Angular.js), a RESTful back-end (via Akka and the Play Framework), and non-blocking asynchronous I/O (via ReactiveMongo). System Architecture

ShockTrade uses a modern Java-based distributed architecture to provide a high performance, highly available, and horizontally scalable system.

The system makes use of modern virtualization technology (Oracle VirtualBox) and currently makes use of three physical quad-core servers to provide the 11 virtual machines depicted above.

The Storage and Reporting Engine

ShockTrade uses a collection of open source systems to provide its complex array of services.

  • MongoDB is the primary storage system. MongoDB (derived from the word "humongous") is a scalable, high-performance, open source NoSQL database, which provides ShockTrade with the ability to provide massive scale from a data perspective.
  • Hadoop, a distributed processing system for large data sets across clusters of computers, provides the platform for the system's reporting engine.
Why MongoDB?

In addition to massive scale, MongoDB has an active developer community and provides an elegant JavaScript/JSON-based query language that offers the ability to quickly produce complex data sets in real-time. Consider the following query, which provides the top 10 stocks with the highest the year-to-date change:

	db.TradingHistory.aggregate( [
		{ $match : { "tradeDate": { $gte : ISODate( "2013-01-01T07:00:00Z" ) } } },
		{ $sort: { tradeDate: 1 } }, 
		{ $group : {
			_id : "$symbol", 
			start_price : { $first : "$close" },
			end_price : { $last : "$close" }
		} },	
		{ $project : {
			_id : 1,		
			change_amt : { $divide : [ { $subtract : [ "$end_price", "$start_price" ] }, "$end_price" ] }
		} },
		{ $sort: { change_amt: -1 } },
		{ $limit : 10 }
	] )

The simple statement above is used to produce a list of the highest performing stocks (year-to-date).

Service-Oriented Architecture

The system currently offers three distinct user experiences, which utilize REST Webservices (XML and JSON) to communicate with the server component that provides company information, stock quotes and statistics:

  • A JFC/Swing Thick Client — offers Simulation mode, which allows a user to test their investment strategies without risking actual capital.
  • An Internet-facing Web Application (HTML5, CSS3, AJAX, Webservices)
  • A CLI Client Application — supports a custom scripting language called QuoteScript, which allows data to be rapidly queried, analyzed and turned into actionable information.

Data Processing

ShockTrade utilizes a custom XML-driven ETL/web-crawling solution (written entirely in Scala) that processes company information, financial statements and stock quotes from a multitude of sources (CSV/flat files, HTML pages and JSON and XML REST web services) and produces reporting and trend analysis.
